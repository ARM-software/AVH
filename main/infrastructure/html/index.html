Infrastructure Service and Tool Integration
Infrastructure

The Arm Virtual Hardware Targets can be accessed via:

  • AWS Marketplace : access to a ready-to-use image on a Cloud Service for CI/CD and MLOps workflow automation.
  • Local Tool Installation : development of test cases and analyzing test failures on a desktop development system.

AWS Marketplace

Arm provides on AWS Marketplace a ready-to-use Amazon Machine Image (AMI) with Arm Virtual Hardware Targets, various compilers, other tools, and resources. The tools are executed in a Linux Virtual Machine and allows you to use the Arm Virtual Hardware environment in various different environments.

The following AWS tutorials help you to get started:

  • How do I create and activate a new AWS account?
  • Tutorial: Get started with Amazon EC2 Linux instances

Subscribe Arm Virtual Hardware

Before you can use the AMI with Arm Virtual Hardware Targets you should have a AWS account. The following steps describe how to subscribe to the AWS Service for Arm Virtual Hardware:

  • Open AWS Marketplace and search for Arm Virtual Hardware.
  • Click on Arm Virtual Hardware to open the product page.
  • Click on Continue to Subscribe to active the product.
  • Click on Configure this software and select the parameters. Note that this page gives you access to the Ami Id.
  • Click on Continue to Launch to active the Arm Virtual Hardware. You have various ways to use the AMI as described in the next section.

Use AMI

The AMI provides a lot of flexibility and gives you the option to integrate Arm Virtual Hardware various CI/CD DevOps environments that range from on-premise IT to Cloud-native infrastructure. The following sections give you some flavors of possible integrations:

  • Run AMI on Local Host allows to use the AMI from a local computer.
  • Run AMI with GitHub Actions explains how to integrate the Arm Virtual Hardware AWS AMI in GitHub action workflows.

Local Tool Installation

Tools on a local PC can be used together with Arm Virtual Hardware for developing and debugging embedded applications.

  • Debug with Arm Development Studio explains how to connect to Arm Virtual Hardware AMI from a local PC and debug the program with Arm DS.
  • Run with MDK-Professional explains how to use Keil MDK for developing and debugging a program with Arm Virtual Hardware Targets running on a local PC.
