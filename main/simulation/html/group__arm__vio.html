Virtual Input/Output (VIO)
Virtual Peripherals

Controls simple I/O such as LED and switches.

Controls simple I/O such as LED and switches.

The Virtual I/O Interface (VIO) implements the CMSIS-Driver VIO for Arm Virtual Hardware Targets.

It consists of the following parts:

  • arm_vio.h: defines platform-specific memory mapping to VIO access structure. It is provided as part of device support files. See in Using Arm Virtual Hardware Targets.
  • ./interface/vio/vio_vht.c: implements CMSIS-Driver VIO variant for Arm Virtual Hardware based on the VIO structure defined in arm_vio.h. It is provided as part of the VHT pack.

Setup

Instructions below cover specifically VIO setup in a project on Arm Virtual Hardware Target.

  • Install following software packs in your development environment, if not available yet:
    • ARM::CMSIS pack for access to CMSIS-Driver VIO API header (cmsis_vio.h)
    • Pack with target device support for access to arm_vio.h
    • Arm VHT pack for access to vio_vht.c.
    The packs can be found on CMSIS Packs page.
  • Add to the project the CMSIS-Driver:VIO (API) component that contains vio_vht.c file.

Usage example

After VIO setup in the project, the interface can be fully used as described in CMSIS-Driver VIO documentation.

For example to control virtual LEDs:

#include "cmsis_vio.h" // ::CMSIS Driver:VIO
int main (void) {
vioInit();
vioSetSignal(vioLED0, vioLEDon);
// ...
vioSetSignal(vioLED0, vioLEDoff);
}

Template header file

Below is a reference header file arm_vio.h that defines Virtual Input/Output (VIO) mapping.

The actual implementation can vary depending on the target architecture and is typically provided with the device support files for the target VHTs.

/*
* Virtual Input/Output (VIO)
*/
#ifndef __ARM_VIO_H
#define __ARM_VIO_H
#ifdef __cplusplus
extern "C" {
#endif
#ifndef __IM
#define __IM volatile const /*! Defines 'read only' structure member permissions */
#endif
#ifndef __OM
#define __OM volatile /*! Defines 'write only' structure member permissions */
#endif
#ifndef __IOM
#define __IOM volatile /*! Defines 'read/write' structure member permissions */
#endif
#include <stdint.h>
/// Structure type to access the virtual input/output.
typedef struct
{
/// Signal Output
struct {
__IOM uint32_t mask; /*!< Bit Mask. Offset: 0x0000 (R/W) */
__OM uint32_t signal; /*!< Signal Value. Offset: 0x0004 (-/W) */
} SignalOut;
/// Signal Input
struct {
__IOM uint32_t mask; /*!< Bit Mask. Offset: 0x0008 (R/W) */
__IM uint32_t signal; /*!< Signal Value. Offset: 0x000C (R/-) */
} SignalIn;
__IOM int32_t Value[64]; /*!< Value (32-bit). Offset: 0x0010 (R/W) */
} ARM_VIO_Type;
/* Memory mapping of VIO peripheral */
#define ARM_VIO_BASE (0x4FEF0000UL) /*!< VIO Base Address */
#define ARM_VIO ((ARM_VIO_Type *)ARM_VIO_BASE) /*!< VIO struct */
#ifdef __cplusplus
}
#endif
#endif /* __ARM_VIO_H */
