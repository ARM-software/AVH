◆ SENSOR_EVENT_DATA

#define SENSOR_EVENT_DATA   (1UL << 0)

Data available.

Triggered to indicate that new data is available.

If sensorConfig_t.dma_mode is set to 1 (DMA mode) the event will be generated on sensorConfig_t.fifo.sample_interval.

If sensorConfig_t.dma_mode is set to 0 (FIFO mode) the event will be generated on sensorConfig_t.dma.block_interval.

◆ SENSOR_EVENT_OVERFLOW

#define SENSOR_EVENT_OVERFLOW   (1UL << 1)

Overflow detected.

Triggered to indicate that overflow of data buffer has occurred.
